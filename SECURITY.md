# Security Policy

## Supported Versions

Peak District Downhill is a static, client-side game with no backend and no
user accounts or stored personal data. There is no long-term support branch —
only the version currently deployed at
[joe-heffer.github.io/Peak-District-Downhill](https://joe-heffer.github.io/Peak-District-Downhill/)
(built from the latest commit on `main`) receives security fixes.

| Version         | Supported          |
| ---------------- | ------------------ |
| Latest (`main`)  | :white_check_mark: |
| Older releases   | :x:                |

## Reporting a Vulnerability

Please report security issues privately rather than in a public GitHub issue.

Use [GitHub's private vulnerability reporting](https://github.com/Joe-Heffer/Peak-District-Downhill/security/advisories/new)
for this repository (Security tab → "Report a vulnerability"). If that isn't
available, email jheffer@gmail.com with details.

Please include:

- A description of the issue and its potential impact.
- Steps to reproduce, or a proof-of-concept.
- Any affected file paths or URLs.

You can expect an initial response within 7 days. If the report is confirmed,
a fix will be prioritized and released as soon as practical, with credit given
to the reporter (unless anonymity is requested). If declined, you'll receive
an explanation of why.
